Exploring Layers and Ripples: Studying English with Computer Parsing

Alastair Butler

Hirosaki University



This book introduces computer parsing of English, with an emphasis on how this can help with the goal of studying English. It provides Definite Clause Grammar (DCG) definitions that are executable code for the XSB Tabling Prolog system. These definitions create a parsing system with a wide coverage of English. The system gives feedback about the acceptability and also use of the English that we try to parse.

Chapter 1
Introduction


1.1   Encircling layers

An utterance of English is like an onion with encircling layers around internal elements, as pictured in Figure 1.1.

Figure 1.1: Layers around an innermost element

   

The action of PARSING amounts to: (i) pealing back layers to reach internal elements, and then (ii) placing everything back together again. An innermost element is either the head word of a phrase layer or the verb of a clause layer. While sticking things back together again, we can add labels to keep track of what was revealed when we pealed back the layers. Subsequently, these labels act as handles for accessing the structural analysis revealed by parsing.

    Furthering the onion analogy, the kind of layering seen in Figure 1.2 occurs when there is coordination, with each conjunct having to conform to whatever is possible at the same particular layer.

Figure 1.2: Parallel layers

   
   

    In addition to structures that arise from multiple layers, we will observe ripple effects, where what happens at one layer goes on to have consequences for what is possible in other layers.


1.2   Phrase Structure Rules

Phrase structure rules are a way to build labelled layers around embedded elements. A phrase structure rule has: (i) a left-hand side that is the label for the layer, and (ii) a right-hand side made of one, two, or more embedded elements, where an embedded element can be either the label of a layer or a terminal element. For example, the five rules of (1.1) involve labelled layers: vp, np_sbj, and ip; and terminal elements: [smiles], [and], and [he].

(1.1)
1.  vp --> [smiles].
2.  vp --> vp, [and], vp.
3.  np_sbj --> [he].
4.  ip --> np_sbj, vp.
5.  ip --> ip, [and], ip.

    By following the rules of (1.1), we can assemble the structure of (1.2): We can start with rule 4 where an ip layer can consist of an np_sbj layer followed by a vp layer. With rule 3 we get to a complete np_sbj with the terminal word [he]. With rule 2 we get to a vp, which is itself made up of a vp, followed by an [and] word, followed by a vp. Finally, with rule 1 we can get to complete vp layers with the terminal word [smiles].

1.3   Orientation

Computer parsing in this book will use Definite Clause Grammar notation (DCG; Pereira, and Warren 1980). This is a notation for writing phrase structure grammar rules in which labels for layers take attributes with values unified by Prolog-style term unification. DCG notation is a feature of virtually all Prolog systems. With DCG notation, we can write a set of phrase structure rules and get an executable Prolog program.

    We will want to write phrase structure grammar rules with left recursion, like rules 2 and 5 of (1.1). When there is left recursion, avoiding infinite loops from rule evaluation requires remembering what has been evaluated. This is possible with tabling, a feature of some Prolog systems, notably, the XSB Tabling Prolog system.

    We will proceed as follows. The remainder of this chapter, section 1.4, introduces rules for utterance layers. This will introduce both writing DCG rules and using the Prolog system to query English word input. Subsequent chapters introduce rules to build up a wide coverage of consequences from word input. This starts in chapter 2 with an inventory of all the kinds of words there are in English. Once we have a handle on the range of words, we will want to start putting words together to create phrases, and this is the goal of chapter 3. With access to phrases and words, we will have content for clause layers. Establishing what is possible at the clause layer is the goal of chapter 4. Chapter 5 builds on all the preceeding chapters by integrating clause subordination.


1.4   Utterance layers

The utterance layer is the topmost structural layer of an utterance. The Prolog code of (1.4) defines this layer with a series of phrase structure rules, each of which imposes its own conditions on what needs to be present to say there is content for an utterance layer.

(1.4)
utterance(Id,n(Id,IP,none)) -->
  clause_top_layer(IPl-L1),
  punc(final,L1-[]),
  {
    IP =.. ['IP-MAT'|IPl]
  }.
utterance(Id,n(Id,'CP-QUE'(IP,PU),none)) -->
  clause_top_layer(IPl-[]),
  punc(final_question,[PU]-[]),
  {
    IP =.. ['IP-SUB'|IPl]
  }.
utterance(Id,n(Id,IP,none)) -->
  clause_top_layer(IPl-[]),
  {
    IP =.. ['IML'|IPl]
  }.
utterance(Id,n(Id,'FRAG'(ADJP),none)) -->
  adjective_phrase([ADJP]-[]).
utterance(Id,n(Id,'FRAG'(Phrase),none)) -->
  adverbial_phrase([Phrase]-[]).

    The first rule of (1.4) will create a structure with IP-MAT as the topmost node, given that the content to be parsed involves a list of items where all but the last item satisfies one of the clause_top_layer phrase structure rules (see section 4.3 below), and where the last list item is an instance of final punctuation (see section 2.14 below).

    In Prolog, lists are written with square bracket notation: [a1, ..., aN], where a1 through to aN are the list items.

    In (1.5), a series of Prolog calls are made. The first call has tphrase_set_string to establish a list of items to parse. Let's call this the parse list. The second call has tphrase to question whether the established parse list has content to satisfy the utterance rules of (1.4). If tphrase succeeds then a parse result with ex1 as its ID will be the content of _X. The third call of pp_ptree pretty prints the _X content as bracketed tree output.

(1.5)
| ?- tphrase_set_string(['PRO'('he'),'VBP'(smiles),'PU'('.')]), tphrase(utterance(ex1,_X)), pp_ptree(_X).

( (IP-MAT (NP-SBJ (PRO he))
          (VBP smiles)
          (PU .))
  (ID ex1))

yes

The pretty print from (1.5) shows the return of a structure with IP-MAT (declarative matrix clause) as the topmost node, from which it follows that the first rule of (1.4) completed successfully.

    Now consider (1.6). This involves almost the same parse list as (1.5), only with the last list item (the punctuation: 'PU'('.')) absent.

(1.6)
| ?- tphrase_set_string(['PRO'('he'),'VBP'(smiles)]), tphrase(utterance(ex2,_X)), pp_ptree(_X).

( (IML (NP-SBJ (PRO he))
       (VBP smiles))
  (ID ex2))

yes

The pretty print from (1.6) shows the return of a structure with IML (intermediate clause layer) as the topmost node: The first rule of (1.4) will have failed, but the third rule succeeds. These rules are very similar: Both call the clause_top_layer phrase structure rules, but the third rule sends all of the parse list content to its clause_top_layer call, and so lacks the requirement for the last item of the parse list to be sentence final punctuation.

    As a further example of checking if there is parse list content for an utterance layer, consider (1.7).

(1.7)
| ?- tphrase_set_string(['NS'('boys')]), tphrase(utterance(ex3,_X)), pp_ptree(_X).

( (FRAG (NP (N boys)))
  (ID ex3))

yes

The pretty print from (1.7) shows the return of a structure with FRAG (sentence fragment) as the topmost node. This will have been achieved with the fourth rule of (1.4), which expects the parse list to provide content for the success of one of the adverbial_phrase rules of section 5.5 below.

    To sum up, this section has shown how to query whether a parse list contains content for an English utterance. With success, returned structure tells us about the kind of utterance: declarative matrix sentence (IP-MAT), interrogative sentence (CP-QUE), intermediate clause utterance (IML), or sentence fragment (FRAG).

Chapter 3
Phrase layers


3.1   Introduction

This chapter considers phrase layers. These are intermediate levels of organisation between a word and the clause. Each phrase is made up of a HEAD word and any DEPENDENTs. The head word is the word that the phrase is required to have in order to exist, and may be the only element of the phrase. A phrase is named after the word class of the head word. There are four main phrase types:

  • noun phrases with a noun head word (section 3.2),
  • adverb phrases with an abverb head word (section 3.3),
  • adjective phrases with an adjective head word (section 3.4), and
  • preposition phrases with a preposition head word (section 3.5).

3.2   Noun phrases

This section considers noun phrases. A bare noun phrase is created with (3.1) which calls noun_phrase_top to create list content (NPl). There is a check (NPl \= ['PU'(_)|_]) to ensure that collected content doesn't have punctuation as its first element. Operation (NP =.. ['NP'|NPl]) turns the collected elements into daughter elements for a tree structure represented with a prolog structure that has principal functor 'NP'. This assembled structure is added to a difference list that is a collection point for subsequent content outside the noun phrase.

(3.1)
noun_phrase([NP|L]-L) -->
  noun_phrase_top(NPl-[]),
  {
    NPl \= ['PU'(_)|_],
    NP =.. ['NP'|NPl]
  }.

    A call of noun_phrase_top, (3.2), concerns a difference list (captured with L) that gathers content from either a call of noun_phrase_initial_layer, or a call of noun_head_full.
