Building tree annotation with a spreadsheet

Alastair Butler

Hirosaki University



A spreadsheet consists of a table of cells arranged into columns and rows. The columns are normally represented by letters, “A”, “B”, “C”, etc., while the rows are normally represented by numbers, “1”, “2”, “3”, etc. A single cell can be referenced by its column and row (A5 would represent the cell containing the value jumped in table (2) below). Additionally, spreadsheets have the concept of a range, a group of cells, normally contiguous. For example, the first ten cells in the first column of (2) form the range “A1:A10”, which is filled with the words of the example sentence (1). Punctuation is treated like a word, where each consecutive word fills a single cell of a distinct consecutive row. Cell B11 contains an ID for the sentence, with there needing to be at least one completely blank row (row 12 in (2)) following the row of an ID cell.

(1)
The quick brown fox jumped over the lazy dog.

    The completed tree information of (14) can be represented in traditional bracketed (Penn/CorpusSearch) format as (15). Note that the bracketing keeps consecutive constituents with the same node tag distinct, so that the node indexing introduced in (6) is no longer required.

(15)
( (IP-MAT (NP-SBJ (D The)
                  (ADJP (ADJ quick))
                  (ADJP (ADJ brown))

          (PP-DIR (P-ROLE over)
