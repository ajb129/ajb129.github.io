Grammar and Beyond 3B by Laurie Blass, Susan Iannuzzi, Alice Savage, and Randi Reppen is a textbook published by Cambridge University Press. The Treebank Semantics Parsed Corpus (TSPC) is a corpus of English with hand-worked analysis for half-a-million words. The purpose of this page is to describe TGrep2 search patterns that connect these two resources.

Unit 14
Negative Questions and Tag Questions


14.1   Negative questions

To find negative questions, you can use pattern (14.1):

(14.1)
/^(MD|HVD|HVP|BED|BEP|DOD|DOP)/=aux
  $.. NEG
  $.. /SBJ/
  >> (/^IP-SUB/ > (/^CP-QUE/ !> PRN) !<< (/^IP\b/ << =aux))
  !$,, (__ << (/^W/ < __))

An auxiliary verb is either:

(i)
a modal (MD), or
(ii)
a finite form of HAVE (HVD|HVP), BE (BED|BEP), or DO (DOD|DOP).

Pattern (14.1) says to find an auxiliary verb that is a preceding sister of ($..) both a negation (NEG) and a subject (/SBJ/). Additional constraints say that the auxiliary should have no other intervening clause layer (IP) between it and an IP-SUB that is dominated by a CP-QUE not dominated by PRN. Also, the auxiliary should not be following on from a sister ($,,) that contains a WH word (/^W/ < __). The order between negation and the subject is left unspecified.

    You can specify the order between negation and the subject, so that (14.2) finds negative questions where the subject needs to be a following sister of ($,,) the occurrence of negation.

(14.2)
/^(MD|HVD|HVP|BED|BEP|DOD|DOP)/=aux
  $.. NEG=neg
  $.. (/SBJ/ $,, =neg)
  >> (/^IP-SUB/ > (/^CP-QUE/ !> PRN) !<< (/^IP\b/ << =aux))
  !$,, (__ << (/^W/ < __))

By contrast, (14.3) finds negative questions where the subject needs to be a preceding sister of ($..) the occurrence of negation.

(14.3)
/^(MD|HVD|HVP|BED|BEP|DOD|DOP)/=aux
  $.. NEG=neg
  $.. (/SBJ/ $.. =neg)
  >> (/^IP-SUB/ > (/^CP-QUE/ !> PRN) !<< (/^IP\b/ << =aux))
  !$,, (__ << (/^W/ < __))

    Thus, pattern (14.1) will return both of the trees of (14.4) and (14.5), while (14.2) will return (14.4), and (14.3) will return (14.5).

(14.4)

Did n’t they have to go somewhere ?


        [149_texts_LooseChange@3,1,2,5,7]
(14.5)

did she not go then


        [332_christine_T39@3,1,2,5,8]

14.2   Tag questions

To find tag questions, you can use pattern (14.6):

(14.6)
/^PRN/ > /^IP-MAT/ < (/^CP-QUE/ < (/^IP-SUB/ <-1 /^NP-SBJ/))

This looks for a PRN node that is itself immediately under an IP-MAT, and that has immediately under it a CP-QUE. Furthermore the CP-QUE needs to immediately dominate an IP-SUB that has as its last child an NP-SBJ. Thus, (14.6) returns example sentences like (14.7).

(14.7)

I_mean , Sirena last night was a marvel , was n’t she ?


        [33_texts_TED_talk_11@24,1,25,26,31]

14.3   Utterances with end reactions

Pattern (14.8) finds utterances with end reactions. Are these similar to tag questions?

(14.8)
/^PRN\b/ < /^REACT\b/ > /^IP\b/




Unit 15
That Clauses

That clauses can be found with pattern (15.1). This searches for nodes with a label that starts with CP-THT. Moreover, a found node needs to dominate at least two further node levels. This will find all contentfull that clause instances in the searched data.

(15.1)
/^CP-THT\b/ < (__ < __)

    That clauses may occur with an overt complementizer, which is essentially always the word that. Instances of complementizers are found with pattern (15.2).

(15.2)
/^C\b/ < __

For example, (15.3) is found with pattern (15.2).

(15.3)

The high quality of the seam on Milos ensured that the area remained popular with early travelers .


        [8_texts_ANC_HistoryOfGreece@28]

    That clauses may also occur without an overt complementizer. Such cases are found with pattern (15.4).

(15.4)
/^CP-THT/ < (__ < __) !<< (/^C\b/ < __)

Unit 16
Embedded clauses with Wh- Words and If/Whether


16.1   Free relatives

Free relatives (also known as headless relative clauses) are clauses with wh- words that have the same distribution as complete noun phrases (so they can act as subjects, direct objects, complements of preposition phrases, etc.).

    You can find all of the free relatives with pattern (16.1). Notably, (16.1) has tree relation ‘<:’ to state that the IP-REL node is the only daughter of a found NP node that is a free relative. The other conditions of the pattern concern the need for the free relative itself to contain a wh- (relative) word without there being a distinct intervening IP layer.

(16.1)
/^NP\b/ <: (/^IP-REL/=i << (/^(RPRO|RD|RADV)\b/ < __ !>> (/^IP\b/ >> =i)))

For example, (16.2) is found with pattern (16.1).

(16.2)

What they are after is the beatific vision .


        [28_susanne_G13@2,3,5]

16.2   Embedded wh questions

Pattern (16.3) will find embedded wh- questions that are selected arguments of verbs.

(16.3)
/^CP-QUE-/ < (/^IP/=i << (/^W/ < __ !>> (/^IP\b/ >> =i)))

For example, (16.4) is found with pattern (16.3).

(16.4)

We want you to Know why your support of Goodwill is so important .


        [4_texts_ANC_110CYL067@15,16,18]

Pattern (16.5) will find embedded wh- questions that are complements of preposition phrases.

(16.5)
/^CP-QUE\b/ > /^PP\b/ < (/^IP\b/=i << (/^W/ < __ !>> (/^IP\b/ >> =i)))

For example, (16.6) is found with pattern (16.5).

(16.6)

Could you give me some suggestions for what we could do in your area for about three hours .


        [14_fce_0100_2001_06@21,22,24]

Pattern (16.7) will find embedded questions that are infinitive clauses.

(16.7)
/^CP-QUE\b/ > /^PP\b/ < (/^IP-INF/=i << (/^W/ < __ !>> (/^IP\b/ >> =i)))

16.3   Indirect questions with If/Whether

WQ is the word class tag for if/whether when used as markers of indirect questions. Pattern (16.8) finds instances of if/whether, and so finds instances of indirect questions.

(16.8)
/^WQ\b/ < __

To restrict the search to finding those indirect questions with if, use pattern (16.9).

(16.9)
/^WQ\b/ < If|if

For example, (16.10) is found with pattern (16.9).

(16.10)

Excuse_me_. I was wondering ___if you had a blue purse like this with a longer strap .


        [1_texts_toeic@14,15]

To restrict the search to finding those indirect questions with whether, use pattern (16.11).

(16.11)
/^WQ\b/ < Whether|whether

For example, (16.12) is found with pattern (16.11).

(16.12)

12. Somewhat strikingly , it remains genuinely unclear ___whether the Defendant disputes that proposition .


        [4_texts_written_case_for_sir_john_major@26,27]

To restrict the search to finding indirect questions with if/whether and coordination, use pattern (16.13).

(16.13)
/^WQ\b/ > (/^IML/ $ /^CONJP\b/)

To restrict the search to finding questions with whether that are complements of preposition phrases, use pattern (16.14).

(16.14)
/^WQ\b/
  [   > (/^IP\b/ > (/^CP-QUE\b/ > /^PP\b/))
    | > (/^IML/ > (/^IML/ > (/^IP\b/ > (/^CP-QUE\b/ > /^PP\b/)))) ]

To restrict the search to finding indirect questions with whether that are infinitive clauses, use pattern (16.15).

(16.15)
/^WQ\b/ [ > /^IP-INF\b/ | > (/^IML/ > (/^IML/ > (/^IP-INF\b/))) ]




Unit 17
Direct Speech and Indirect Speech


17.1   Direct speech

Pattern (17.1) will find instances of direct speech where the quoted material is itself a single sentence.

(17.1)
/^IP-MAT/ $, PUQ

For example, (17.2) is found with (17.1).

(17.2)

“ Hi - I ’m calling to make a reservation , ” the voice , a woman , begins .


        [18_texts_Duplex@4,2]

Pattern (17.3) will find instances of direct speech where the quoted material is a series of sentences.

(17.3)
/^multi-sentence/ $, PUQ

For example, (17.4) is found with (17.3).

(17.4)

He said now , “ I ’ve got the perfect headquarters set up . The old Haskell mine ” .


        [93_susanne_N07@14,12]

Pattern (17.5) will find instances of direct speech where the reporting clause occurs in the middle of the direct speech.

(17.5)
PRN < /^IP-MAT/ > /^(IP|IML)/ $ PUQ

For example, (17.6) is found with (17.5).

(17.6)

        [173_texts_LooseChange@19,1,2,20]

17.2   Indirect speech

Indirect speech is the content of a that clause (with optional that word) that is selected by a reporting verb. Pattern (17.7) will find instances of indirect speech where the reporting verb is SAY.

(17.7)
/^V/ < /^(say|said)/ $ /^CP-THT-OB1/

Note how the regular expression /⁁(say|said)/ of (17.7) will match the different forms of SAY: say, says, saying, and said.

    For example, (17.8) is found with (17.7).

(17.8)

They ___say the company has produced some shoddy work and charges too much money .


        [14_texts_Voice_of_America@5,6,7]



Unit 19
The Passive (1)

Pattern (19.1) finds all occurrences of passives in the searched data. This looks for a constituent tag that is under an IP-PPL projection and that contains LGS, which stands for logical subject.

(19.1)
/LGS/ < __ > /^IP-PPL/

The logical subject node found with (19.1) might dominate a zero element (‘*’) to give a passive clause where the logical subject is not overtly present. An example of this is (19.2).

(19.2)

An important early material , obsidian , was discovered on the island of Milos .


        [6_texts_ANC_HistoryOfGreece@24]

The logical subject node found with (19.1) might form a PP constituent, typically introduced with by. An example of this is (19.3).

(19.3)

He was followed in 336 b.c. by his son Alexander the Great , one of the most remarkable leaders in history .


        [56_texts_ANC_HistoryOfGreece@19]

19.1   Present and past forms of the passive

  • The simple present form of the passive involves having the present form of BE followed by the past participle form of the main verb. Relevant examples can be found with pattern (19.4).
(19.4)
/LGS/ < __ > (/^IP-PPL-CAT/ $ BEP)
  • The present perfect form of the passive involves having a present form of HAVE followed by the past participle form of BE (been) followed by the past participle form of the main verb. Relevant examples can be found with pattern (19.5).
(19.5)
/LGS/ < __ > (/^IP-PPL-CAT/ > (/^IP-PPL-CAT/ $ HVP))
  • The present progressive form of the passive involves having a present form of BE followed by the present participle form of BE (being) followed by the past participle form of the main verb. Relevant examples can be found with pattern (19.6).
(19.6)
/LGS/ < __ > (/^IP-PPL-CAT/ > (/^IP-PPL-CAT/ $ BEP))
  • The simple past form of the passive involves having the past form of BE followed by the past participle form of the main verb. Relevant examples can be found with pattern (19.7).
(19.7)
/LGS/ < __ > (/^IP-PPL-CAT/ $ BED)
  • The past perfect form of the passive involves having a past form of HAVE followed by the past participle form of BE (been) followed by the past participle form of the main verb. Relevant examples can be found with pattern (19.8).
(19.8)
/LGS/ < __ > (/^IP-PPL-CAT/ > (/^IP-PPL-CAT/ $ HVD))
  • The past progressive form of the passive involves having a past form of BE followed by the present participle form of BE (being) followed by the past participle form of the main verb. Relevant examples can be found with pattern (19.9).
(19.9)
/LGS/ < __ > (/^IP-PPL-CAT/ > (/^IP-PPL-CAT/ $ BED))

19.2   Passive forms with direct and indirect objects

Some verbs, such as GIVE, OFFER, SHOW, and TELL, can have two kinds of object: a direct and an indirect object. Both kinds of object can be made the subject when the clause is passive.

  • You can use pattern (19.10) to find examples where what would be the indirect object of an active clause is the subject of a passive clause.
(19.10)
/LGS/ $ /^NP-OB1/
  • You can use pattern (19.11) to find examples where what would be the direct object of an active clause is the subject of a passive clause.
(19.11)
/LGS/ $ (/OBU/ < (/^P-ROLE/ < to)) !$ /PRD/

Unit 21
Subject relative clauses

You can use pattern (21.1) to find subject relative clauses.

(21.1)
/SBJ/ < RPRO

Pattern (21.1) finds subject relative clauses by locating a relative pronoun that makes up the content of a subject constituent. A relative pronoun is a word tagged RPRO, e.g. which, who, that. The relative pronoun links the internal contribution of the relative clause to the overall noun phrase contribution of which the relative clause is a part.


21.1   Restrictive subject relative clauses

Relative clauses can be either restrictive or non-restrictive. In written language, restrictive relative clauses are not immediately preceded by punctuation. Pattern (21.2) finds subject relative clauses that are not immediately preceded by punctuation.

(21.2)
/^IP-REL/=rel << (/SBJ/ < RPRO !>> (/^IP-REL/ >> =rel)) !$, PU $ __

For example, (21.3) is found with (21.2).

(21.3)

This is a watch that creates a lifestyle for modern individuals who seek affluence and convenience in their life .


        [5_texts_seiko_9R65@32,33,34]

21.2   Non-restrictive subject relative clauses

In written language, we can find non-restrictive relative clauses by ensuring that the found relative clause is immediately preceded by punctuation. Such a search is accomplished with pattern (21.4).

(21.4)
/^IP-REL/=rel << (/SBJ/ < RPRO !>> (/^IP-REL/ >> =rel)) $, PU

For example, (21.5) is found with (21.4).

(21.5)

Everyone turned to stare at her , except the boy , who just carried on .


        [105_texts_LooseChange@32,30,33,34]

21.3   Subject relative clauses with Whose

So far we have considered cases where there is a subject that consists of a relative pronoun. It is also possible for the subject to contain its own restrictive noun phrase content in addition to a genitive (-GENV) relative pronoun whose. Pattern (21.6) will search for relative clauses that contain subjects that contain genitive relative pronouns.

(21.6)

Unit 22
Object relative clauses

You can use pattern (22.1) to find object relative clauses.

(22.1)
/OB/ [ < RPRO | < /^\*T\*/ ]

Pattern (22.1) finds object relative clauses by locating a constituent with object function that is either made up of a relative pronoun or a zero element.

  • A relative pronoun is a word tagged RPRO, e.g. which, who, whom, that.
  • A zero element is an element which is required by the clause, but has an absence that is construction licensed.

Such objects link the internal object for the relative clause to the overall noun phrase contribution of which the relative clause is a part.


22.1   Restrictive object relative clauses

Like subject relative clauses, object relative clauses can be either restrictive or non-restrictive. In written language, restrictive relative clauses are not immediately preceded by punctuation. Pattern (22.2) finds object relative clauses that are not immediately preceded by punctuation.

(22.2)
/^IP-REL/=rel
  << (/OB/ [ < RPRO | < /^\*T\*/ ] !>> (/^IP-REL/ >> =rel))
  !$, PU
  $ __

For example, (22.3) is found with (22.2).

(22.3)

Every movement she made seemed unnecessarily noisy .

Unit 26

Conjunctions connect words, phrases, and clause layers. They do so by connecting elements of equivalent grammatical ‘status’. For example, nouns phrases can be coordinated with other noun phrases, clauses can be coordinated with other clauses, but you cannot directly coordinate a noun phrase with a clause.

    With the annotation, all instances of coordination involve the projection(s) of CONJP layers of structure. We can search for this structure by searching for CONJP nodes with pattern (26.1).

(26.1)
CONJP
